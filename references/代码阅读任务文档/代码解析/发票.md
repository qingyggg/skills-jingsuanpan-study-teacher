## `InOutSum` 返回字段全图

字段编号不是按"发票类型编号"排的，而是**后端拼凑的展示序号**，一开始看确实迷惑。对照代码逐一列出：

| `inSum6`     | 已认证     | 电专（type=4）           | `rzzt <> 2`                                           |

### 销项（outSum 系列）— 你没看懂的这部分

销项**没有认证状态**的概念（销项是你开出去的，不需要认证），所以只按发票类型分：

| 字段      | 对应发票类型 | 说明                   |
| --------- | ------------ | ---------------------- |
| `outSum`  | 全部         | 所有销项总计           |
| `outSum1` | type=1 专票  | 增值税专用发票         |
| `outSum2` | type=2 普票  | 增值税普通发票         |
| `outSum3` | type=3 无票  | 无票销项（不开票收入） |
| `outSum4` | type=4 电专  | 增值税电子专用发票     |
| `outSum5` | type=5 电普  | 增值税电子普通发票     |

对应代码：

```132:137:api/invoice/invoice.go
"outSum":  outSum,                                                          // 销项总计
"outSum1": invoiceOutSum(comId, period, s_ea.InvoiceOutType1, str, txItem), // 销项增值税专用发票
"outSum2": invoiceOutSum(comId, period, s_ea.InvoiceOutType2, str, txItem), // 销项增值税普通发票
"outSum3": invoiceOutSum(comId, period, s_ea.InvoiceOutType3, str, txItem), // 无票销项
"outSum4": invoiceOutSum(comId, period, s_ea.InvoiceOutType4, str, txItem), // 销项电子专票
"outSum5": invoiceOutSum(comId, period, s_ea.InvoiceOutType5, str, txItem), // 销项电子普票
```

`invoiceOutSum` 的 SQL 就是简单的 `WHERE com_id=? AND invoice_type=?`，不加任何认证状态条件。

## 一句话总结

- **进项**：有认证问题（专票/电专等需认证才能抵扣），所以字段按 **（类型 × 认证状态）** 两个维度展开，加上未认证池这个特殊来源，导致字段多且编号不连续。
- **销项**：没有认证状态，只按发票类型分 5 类，`outSum1~5` 就是 1 专票、2 普票、3 无票、4 电专、5 电普，对应 `InvoiceOutType1~5` 常量。
