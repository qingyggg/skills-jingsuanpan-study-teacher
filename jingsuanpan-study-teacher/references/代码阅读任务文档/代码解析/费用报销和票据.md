## 板块十二：费用报销（`api/fee`）

### 业务场景

员工报销差旅费、餐费、办公耗材等日常费用，会计把这些报销单录入系统，之后一键生成费用凭证。

### 数据结构（两层）

```
EaFee（报销单主表）
  └── EaFeeItem（费用明细行）× N
```

- 一张报销单可以有多条明细（比如一次报销里既有交通费又有餐费）
- 每条明细挂一个**费用科目**（如管理费用-差旅费）和**往来单位**（`CorpId`）
- `SettleAmt`（现金结算金额）：支持部分现金支付，其余挂应付

### 与凭证的关联

```
EaFee.VoucherID → ea_voucher（已生成凭证后回写）
```

一键生成时（`OneKeyVoucherFee`）：

- 借方：各费用科目（按明细行）
- 贷方：应付账款（往来单位）或现金科目（若有现金结算）

### 接口（5个）

| 接口        | 说明                                           |
| ----------- | ---------------------------------------------- |
| `saveFee`   | 新增/编辑报销单（批量，一次可保存多张）        |
| `feeList`   | 列表查询（支持按凭证状态、往来单位、科目过滤） |
| `delFee`    | 删除                                           |
| `importFee` | Excel 导入批量录入                             |
| `feeExport` | 导出                                           |

### 与其他模块的关系

- 凭证生成：走 `OneKeyVoucherFee`（在一键生成凭证模块讲过）
- `VoucherID` 非空 = 已生成凭证，列表里会显示凭证号

## 板块十三：票据管理（`api/bill`）

### 业务场景

企业收到或开出**商业汇票**（应收票据/应付票据），会计把票据信息登记入系统：票面金额、到期日、对方单位、票据科目等。主要用于票据到期提醒和一键生成凭证。

### 数据结构（单表）

```
EaBill（票据主表）
  - Type          收入(1) 还是 支出(0)
  - Amount        票据金额
  - TransDate     到期日
  - SubjectId     结算科目（现金/银行存款等）
  - BillSubjectId 票据科目（应收票据/应付票据）
  - BusinessTypeId 业务类型
  - VoucherId     已生成凭证后回写
```

### 与凭证的关联

一键生成（`OneKeyVoucherBill`）：

- 收票时：借票据科目，贷结算科目
- 付票时：反向

### 接口（3个，比费用模块精简）

| 接口       | 说明                  |
| ---------- | --------------------- |
| `saveBill` | 新增/编辑票据（批量） |
| `billList` | 列表查询              |
| `delBill`  | 删除                  |

---

## 两个模块的共同特点（快速理解）

这两个模块在整个系统里是**「数据暂存区」**——本质上都是：

```
业务单据录入（费用单 / 票据）
  ↓ voucher_id 为空 = 待生成凭证
一键生成凭证（OneKeyVoucher type=fee/bill）
  ↓ voucher_id 有值 = 已生成凭证
凭证 → 科目余额 → 财务报表
```

和进项发票、银行流水、现金收支是**同一套流程**，都是作为凭证的「来源单据」存在，业务单据本身不直接影响科目余额，必须通过凭证才能进入账务体系。
